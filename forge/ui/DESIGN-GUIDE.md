# Forge UI/UX Master Design Guide

Saved from thority directive, 2026-03-08.
Canonical design bible for all Forge UI work.

## ORYN Aesthetic Direction
- Brutalist-tech: sharp corners (0-2px radii), crosshair registration marks
- Section numbering (01-08) on page headers
- 0.15em+ letter-spacing on uppercase labels
- Film grain SVG noise overlay at 0.03 opacity
- Zero shadows - flat hover states via border-color only
- 4px clinical status dots, 1px borders throughout
- Technical index nav with construction-line hairlines
- Accent: #D4382C (cooler ORYN red)
- Reference: Linear/Vercel/Raycast aesthetic family

## Core Design Rules
- Simplicity is architecture - every element must justify its existence
- Consistency is non-negotiable - same component, same look everywhere
- Hierarchy drives everything - one primary action per screen, unmissable
- Alignment is precision - every element on a grid, no exceptions
- Whitespace is a feature - breathing room feels premium
- Design the feeling - calm, confident, quiet
- Responsive is the real design - mobile first
- No cosmetic fixes without structural thinking

## Jobs Filter
For every element: Would a user need to be told this exists? Can this be removed? Does this feel inevitable? Say no to 1,000 things.

## Scope
Touch: visual design, layout, spacing, typography, color, interaction, motion, accessibility
Do NOT touch: app logic, state management, API calls, data models, backend

## Audit Dimensions
Visual hierarchy, spacing/rhythm, typography, color, alignment/grid, components, iconography, motion/transitions, empty states, loading states, error states, dark mode, density, responsiveness, accessibility
